# Parasite SEO Content Guidelines

## Overview
This document provides guidelines for creating high-quality parasite SEO content that drives traffic to WebPify tools while providing genuine value.

## Core Principles

### 1. Value First
- Provide real, actionable information
- Answer user questions comprehensively
- Include practical examples and results
- Never create thin or doorway pages

### 2. Natural Integration
- CTA should feel natural, not forced
- Tool mention should add value to content
- Don't over-promote or spam
- Use "compared to" or "I recommend" language

### 3. Platform Compliance
- Follow each platform's community guidelines
- No keyword stuffing or manipulation
- Authentic, helpful content only
- Engage with comments genuinely

## Keyword Targets (Priority Order)

### Tier 1 (High Volume, High Intent)
1. "compress png online" – Intent: Reduce PNG file size
2. "convert image to webp" – Intent: Modern format conversion
3. "jpeg image compressor" – Intent: Compress photos

### Tier 2 (Medium Volume, Specific Intent)
4. "png file size reducer" – Intent: Smaller PNG files
5. "webp vs png" – Intent: Format comparison
6. "best image format for seo" – Intent: SEO optimization

### Tier 3 (Long-tail, Niche)
7. "how to compress png without losing quality"
8. "why are my images so large"
9. "reduce image file size for website"
10. "png to webp converter online free"

## Platform-Specific Guidelines

### Medium
- **Length:** 800-1200 words
- **Tone:** Professional, informative
- **Structure:** Clear headings, short paragraphs
- **Images:** 2-3 relevant images or screenshots
- **CTA:** Natural integration in "Best Tool" section
- **Tags:** webdev, tutorial, performance, seo, images

### Dev.to
- **Length:** 600-1000 words
- **Tone:** Developer-friendly, technical but accessible
- **Structure:** Bullet points, code blocks (if relevant), headings
- **Images:** Optional but helpful
- **CTA:** Casual recommendation style
- **Tags:** webdev, tutorial, performance, seo
- **Engagement:** Respond to comments

### Hashnode
- **Length:** 800-1500 words
- **Tone:** Technical, detailed, blog-style
- **Structure:** Clear sections, numbered lists, examples
- **Images:** Professional screenshots or diagrams
- **CTA:** Strong recommendation with reasons
- **Tags:** web-development, performance, image-optimization
- **Series:** Consider creating a series on image optimization

### Reddit
- **Length:** 400-800 words
- **Tone:** Conversational, community-focused, honest
- **Structure:** TL;DR at top, clear sections, lists
- **Language:** "I found", "I recommend", "In my experience"
- **CTA:** Subtle, authentic recommendation
- **Subreddits:** r/webdev, r/web_design, r/SEO, r/Frontend
- **Rules:** NO direct promotion, provide value first
- **Engagement:** MUST respond to comments

### GitHub
- **Format:** README.md style
- **Length:** 400-600 words
- **Tone:** Technical, documentation-style
- **Structure:** Badges, features, quick start, FAQs
- **Repository:** Can be a simple tool guide or resource
- **CTA:** Clear link in Quick Start section
- **Updates:** Keep updated with current features

## Content Structure Template

### Opening (Hook)
- Start with the problem
- Relate to reader's pain point
- Promise a solution

**Example:**
"Large PNG files slow down your website. Here's how to compress them by 60-90% without quality loss."

### Problem Definition
- Explain why the issue matters
- Include specific consequences
- Use data/statistics if available

**Example:**
- Slow page loads → Higher bounce rates
- Poor SEO rankings → Less traffic
- High bandwidth costs → Wasted money

### Solution Explanation
- How the solution works (briefly technical)
- Benefits of the approach
- Why it's better than alternatives

**Example:**
"PNG compression uses pngquant algorithm to optimize color palettes and remove metadata while preserving visual quality."

### Tool Recommendation
- Natural transition to WebPify
- Specific features and benefits
- Direct link with context

**Example:**
"I use WebPify for PNG compression because it's client-side (private), has no file limits, and achieves 60-90% reduction."

### Practical Guidance
- Step-by-step process
- Expected results
- Tips and best practices

### FAQs
- Address common concerns
- Factual, concise answers
- Build trust and authority

### CTA (Call to Action)
- Clear next step
- Direct link
- Benefit-focused language

**Example:**
"Try compressing your PNG images: [webpify.vercel.app/image/compressor/png](https://webpify.vercel.app/image/compressor/png)"

## Writing Rules

### ✅ DO
- Provide genuine value
- Use natural language
- Include real examples and results
- Answer questions comprehensively
- Engage authentically with comments
- Follow platform guidelines
- Update content periodically
- Cross-link related articles

### ❌ DON'T
- Keyword stuff
- Create thin content
- Spam or over-promote
- Copy/paste exact content across platforms
- Ignore comments
- Use deceptive tactics
- Violate platform TOS
- Compete with main site content

## CTA Best Practices

### Natural Integration
```markdown
❌ "Click here to use our amazing tool now!!!"
✅ "I use WebPify for PNG compression: [link]"

❌ "The best PNG compressor in the world!"
✅ "WebPify compresses PNGs by 60-90% with lossless quality"

❌ Multiple CTAs everywhere
✅ One clear CTA in logical place
```

### Link Formatting
- Always use full URLs with https://
- Use descriptive link text (not "click here")
- Add context around links
- Make links stand out naturally

**Example:**
"Compress PNG images: **[webpify.vercel.app/image/compressor/png](https://webpify.vercel.app/image/compressor/png)**"
